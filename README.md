# PRISM RIFT VANGUARD

Three.js ベースで制作された、キャンペーン進行型の 3D シューティングゲームです。  
各ミッションを突破しながらクリスタルを集め、任務間インターバルのハンガーショップで機体を強化し、最終決戦の制覇を目指します。

> プレイヤー向けの遊び方は [`doc/PLAYER_GUIDE.md`](doc/PLAYER_GUIDE.md) を参照してください。  
> このREADMEは、開発者・改造者・実装確認者向けの内容です。

## 概要

**PRISM RIFT VANGUARD** は、

- ミッション制 3D シューティング
- ミッション間のショップ強化によって機体性能を拡張する進行型構成
- 通常ショットとクールタイム制プラズマショットを軸にした戦闘
- 日本語 / 英語 / 中国語の多言語 UI

を備えた、ブラウザ実行型のキャンペーンシューターです。

現在の実装では、タイトル画面から **チュートリアル** と **キャンペーン開始** を個別に選択できます。  
本編は複数ミッションで構成され、空域ごとに敵構成、演出、地形テーマ、ボスが切り替わります。  
環境ギミックは一部ミッションで有効です。

## 起動方法

### 1. ローカルサーバーを起動

プロジェクトのルートディレクトリで、たとえば次のようにサーバーを立ち上げます。

```bash
python -m http.server 8000
```

### 2. ブラウザで開く

```text
http://localhost:8000/
```

`index.html` が読み込まれ、タイトル画面が表示されます。

## タイトル画面と主要導線

通常プレイ時のタイトル画面では、次の導線を持ちます。

1. **チュートリアル**
2. **キャンペーン開始**
3. **オプション**
4. **レコード**
5. **敵設定資料**
6. **クレジット**

このうち **敵設定資料** は、**最終ミッションのクリア記録が保存されている場合のみ表示**されます。  
初期状態や未クリア状態では非表示です。

## ゲーム内容の要点

### キャンペーン進行

- チュートリアルは独立した開始導線です。
- キャンペーンは最初の実戦ミッションから開始します。
- 各ミッションは複数 WAVE + ボス戦、またはボス直行構成で進行します。

### 成長要素

ミッション間インターバルのハンガーショップでは、クリスタルを消費して機体性能を強化できます。

主な強化項目:

- 最大ライフ
- 装甲
- クリスタル吸引
- 通常ショット威力
- 通常ショット連射性能
- 貫通弾
- 多方向射撃
- ホーミング補助
- プラズマ威力
- プラズマ爆風半径
- プラズマ冷却短縮
- プラズマ多方向化

多方向化のトレードオフは武装ごとに異なります。

- **通常ショットの多方向化**: 連射レート低下あり
- **プラズマの多方向化**: クールタイム増加あり

### 記録保存

タイトル画面の **レコード** では、以下を確認できます。

- ミッションごとのハイスコア
- ミッションごとの最速タイム
- ミッションごとの獲得勲章
- キャンペーン最終結果
- 保存データ削除

## オプション項目

現在の実装で変更できる主な項目は以下の通りです。

### システム
- 言語

### オーディオ
- サウンドテスト
- BGM 音量
- SE 音量

### 操作
- マウス感度
- Y 軸反転

### 映像
- 描画品質 (`LOW / MEDIUM / HIGH`)
- 視野角 (`FOV`)
- 発光 / エフェクト強度 (`標準 / 控えめ / 最小`)

### 照準・戦闘補助
- 照準プリセット
- 照準サイズ
- 被弾方向表示

### HUD
- HUD 不透明度
- ミニマップ表示
- ミニマップ拡大率
- 敵マーカー表示
- 高コントラスト表示

オプション画面はタイトル画面だけでなく、**ポーズ中** と **インターバル中** からも開けます。

## デバッグ機能

デバッグ画面は通常プレイでは非表示です。  
URL に `?debug=1` を付けて起動すると有効になります。

例:

```text
http://localhost:8000/?debug=1
```

デバッグ画面では、たとえば次のような機能が利用できます。

- タイトルから開始するミッションの変更
- GODMODE 切り替え
- BOSS MODE 切り替え
- パフォーマンスレポート確認

実行中のデバッグ操作として、画面内敵の一掃などの補助導線も組み込まれています。

> デバッグ機能は開発・確認用です。プレイヤー向け説明からは分離しています。

## 保存データ

本作はブラウザの **localStorage** を使って、オプション設定とプレイ記録を保存します。

### 保存される主な内容

- オプション設定
  - 言語
  - サウンドテスト選択曲
  - BGM / SE 音量
  - マウス感度
  - Y 軸反転
  - 描画品質
  - FOV
  - エフェクト強度
  - HUD / 照準 / ミニマップ関連設定
- プレイ記録
  - ミッションごとのハイスコア
  - ミッションごとの最速タイム
  - ミッションごとの獲得勲章
  - キャンペーン最終結果

### データ削除

タイトル画面の **レコード** から削除できます。

## プロジェクト構成

```text
index.html
styles.css
assets/
  audio/
doc/
  CREDITS.txt
  PLAYER_GUIDE.md
src/
  app/
  audio/
  combat/
  core/
  data/
  debug/
  enemies/
  entities/
  fx/
  i18n/
  player/
  progression/
  render/
  spatial/
  storage/
  ui/
  utils/
  world/
```

### 構成の目安

- `src/data/` : ミッション、敵、バランス、音声定義などのデータ
- `src/player/` : プレイヤー移動、武装、被弾、演出
- `src/enemies/` : 通常敵、ボス、敵 AI
- `src/progression/` : ミッション進行、インターバル、実績 / 勲章、強化
- `src/ui/` : タイトル、レコード、敵設定資料、オプション、クレジット、HUD
- `src/storage/` : localStorage ベースの設定 / 記録保存
- `src/world/` : 地形、空域テーマ、環境オブジェクト
- `src/render/` : Three.js の描画制御
- `src/debug/` : デバッグ導線と計測補助
- `doc/` : クレジットやプレイヤー向け説明書などの付帯文書
